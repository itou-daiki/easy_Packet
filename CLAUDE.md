
ネットワーク学習Webアプリ「easyPacket」仕様書
1. アプリケーション概要
アプリケーション名:
easyPacket (イージーパケット)

コンセプト:
コマンド操作と視覚的なフィードバックを通じて、ネットワークの基本概念を誰もが楽しく、かつ実践的に学べるWebアプリケーションです。ユーザーが入力したコマンドによって「データ（パケット）がどのようにネットワークの海を旅するのか」をアニメーションで可視化し、直感的な理解を促します。

学習目標:

ping, nslookup, traceroute 等の基本コマンドの役割と使い方を習得する。

IPアドレスとDNSの名前解決、そしてルーティングの仕組みを、体験を通して理解する。

パケット、ルーター、サーバーといった専門用語に慣れ親しむ。

2. アーキテクチャ (GitHub Pages対応)
本アプリは、サーバー側のプログラム（バックエンド）を持たず、HTML, CSS, JavaScriptのみで構成される完全な静的サイトとして構築します。これにより、GitHub Pages上で手軽に公開・運用が可能です。

コマンド実行: ユーザーが入力したコマンドは、ブラウザ上で動作するJavaScriptが解釈し、あらかじめ定義されたシナリオに基づいて結果をシミュレートします。実際のネットワークにパケットを送信することはありません。

データ管理: 演習に必要なドメイン名、IPアドレス、経路情報などは、JavaScriptのオブジェクトやJSONファイル（dns.json, routes.json）としてアプリ内に保持します。

3. 画面構成案
画面は大きく3つのエリアに分割し、学習、実践、可視化をシームレスに行えるUIを提供します。

① 学習・課題エリア（左）	② 実行・可視化エリア（中央）	③ コマンド・用語エリア（右）
チュートリアルや「Webサイトが表示されない！」といったシナリオ形式の演習課題が表示されます。	上部にネットワーク模式図が表示され、コマンド実行に応じてパケットのアニメーションが動きます。 下部にCodemirrorで実装されたインタラクティブ・コンソールが表示されます。	上部によく使うコマンドのボタンが配置され、クリックでコンソールに入力できます。 下部には用語集があり、クリックすると解説が表示されます。

4. 機能仕様詳細 (シミュレーションベース)
① コマンドシミュレーター機能
本アプリの核となる機能です。実際のコマンドの挙動をJavaScriptで忠実に再現します。

内部データベース
シミュレーションの基となる2つのJSONファイルを用意します。

dns.json: ドメイン名とIPアドレスの対応表

JSON

{
  "google.com": "8.8.8.8",
  "github.com": "20.205.243.166",
  "example-school.ac.jp": "192.0.2.1",
  "broken-server.com": "TIMEOUT"
}
routes.json: tracerouteのための経路情報（経由ルーターのリスト）

JSON

{
  "google.com": [
    { "ip": "192.168.1.1", "name": "my-router.local", "time": 1 },
    { "ip": "10.0.0.1", "name": "provider-router-1.isp.net", "time": 15 },
    { "ip": "203.0.113.5", "name": "ix-router.net", "time": 20 },
    { "ip": "8.8.8.8", "name": "google-dns.com", "time": 22 }
  ]
}
実装コマンド
nslookup <ドメイン名>
dns.json を参照し、対応するIPアドレスをコンソールに表示します。同時に、DNSサーバーへ問い合わせるアニメーションを再生します。

ping <ドメイン名>

まず内部的にnslookupの処理を行い、IPアドレスを解決します。

PCから目的地までパケットが一気に飛んでいき、返ってくるアニメーションを再生します。

応答時間(time)やTTLは、リアルに見えるように乱数を加えてコンソールに表示します。

"TIMEOUT"が設定されたドメインには、タイムアウトエラーを返します。

traceroute <ドメイン名> 🚀

routes.json を参照し、経路情報の配列を取得します。

配列内のルーターを1つずつ順番に処理します。

コンソールには、各ルーター（ホップ）の情報が1行ずつ表示されます。

アニメーションでは、コンソールの表示と連動して、パケットが各ルーターをリレー形式で経由していく様子を段階的に描画します。

ipconfig (または whoami)
外部の無料API（例: https://api.ipify.org）を呼び出し、ユーザー自身の実際のグローバルIPアドレスを取得してコンソールに表示します。

② 可視化・演習・UI機能
ネットワーク模式図:

あなたのPC、複数の経由ルーター、ISP（プロバイダ）の雲、DNSサーバー、宛先サーバーなどのアイコンを配置し、パケットの旅を分かりやすく表現します。

インタラクティブなUI:
コンソール出力内の専門用語（TTLなど）にマウスを合わせると解説が表示されたり、コマンドボタンで入力を補助したりすることで、学習者の体験をサポートします。

5. 教育的価値
安全性: 実際のネットワークを使用しないため、どのような環境でも安全に利用できます。

安定性: 外部環境に依存せず、いつでも同じ結果を再現できるため、授業計画が立てやすくなります。

本質的な学習: 「パケットという主役が、IPアドレスという住所を頼りに、ルーターを経由して旅をする」という、ネットワークの根幹をなす仕組みの理解に集中できます。
